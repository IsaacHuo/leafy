# 产品总览

## 产品定位

MyLeafy 是以课表和学业数据为核心的高校生活工具。目前项目主要面向北京林业大学，目标不是简单复制教务网页，而是把分散的高频任务组织为连续流程：登录学校系统、获取并缓存授权数据、管理课表与学习任务、参与校园社区，并在用户明确允许的范围内使用 AI 辅助整理与规划。

## 用户价值

| 场景 | MyLeafy 提供的能力 |
|---|---|
| 今天上什么课 | 周课表、课程详情、重叠课程布局、Widget 与提醒 |
| 学业信息分散 | 成绩、考试、培养方案、校历、空教室与学习空间 |
| 需要协作或分享 | 课表分享图、共享课表、只读授权与撤销关系 |
| 获取校园信息 | 社区、公告、通知、目录与评价能力 |
| 整理复杂信息 | Leafy AI、本机上下文、公开检索、带来源回答与 Artifact |
| 个性化使用 | 外观、主题色、显示密度、多语言和课表背景 |

## 数据权威与所有权

| 数据类别 | 权威来源 | 默认位置 |
|---|---|---|
| 课表、成绩、考试、培养数据 | 学校教务系统 | 设备本地缓存 |
| 备注、提醒、个人学习记录 | 用户 | 设备本地；特定功能可明确同步 |
| 社区内容、通知、共享与评价 | MyLeafy 业务系统 | Supabase |
| AI Key 与本机对话 | 用户设备 | Keychain / 本地存储 |
| 管理操作与审计 | MyLeafy 运营系统 | Supabase，仅授权管理员可读 |

学校密码不应上传到 MyLeafy 业务后端；共享课表只上传用户主动发布的必要字段子集。

## 产品状态

项目已经形成 iOS、Supabase、官网与 Web 管理端的完整工程结构。当前优先级是：

1. 提高教务解析韧性和故障恢复质量。
2. 统一不同校园的适配协议与能力配置。
3. 完善课表、日程和学习空间之间的数据连接。
4. 收紧身份绑定、共享、导出与管理操作的安全边界。
5. 保持实现、测试、D2 图表与公开文档一致。

以上为方向，不构成固定版本或时间承诺。详细内容见仓库中的[项目总览](https://github.com/IsaacHuo/leafy/blob/main/docs/overview.md)与[发展方向](https://github.com/IsaacHuo/leafy/blob/main/docs/roadmap.md)。
