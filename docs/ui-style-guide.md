# MyLeafy UI 风格规范

本文定义 MyLeafy iOS 界面的视觉语言、设计令牌、组件使用、页面模式和可访问性要求。规范以当前 SwiftUI 实现为准，主要代码位于 `leafy/App/Theme/AppTheme.swift` 与 `leafy/App/Theme/AppChrome.swift`。

页面职责和流程见[App 产品设计](app-design.md)。

## 1. 设计方向

MyLeafy 的界面应呈现轻量、原生、安静而有辨识度的校园工具感：

- 以系统控件、系统字体、SF Symbols 和原生导航行为为基础。
- 主题色用于建立连续性，不大面积覆盖内容或替代信息层级。
- 使用留白、背景层级和弱分隔线组织内容，减少厚重阴影和高对比边框。
- 高密度页面首先保证可读性和触控准确性，再考虑装饰效果。
- iOS 26 的 Liquid Glass 是增强层；iOS 17–25 必须有完整、可读的 material/背景回退。
- 深色模式、自定义主题、显示密度和多语言是系统能力，不是发布前临时适配。

## 2. 设计令牌

### 2.1 主题色

`AppThemeColorPreference` 当前提供：

| 选项 | 用途 |
|---|---|
| 鼠尾草绿 | 默认品牌主题，低饱和、自然 |
| 蒂芙尼蓝 | 冷色替代主题 |
| 糖果莓粉 | 高辨识度暖色主题 |
| 自定义 | 用户选择任意 6 位 HEX 颜色，由代码生成强调色、浅底色和课程色阶 |

历史版本中的其他固定颜色会迁移为自定义颜色，不再作为独立预设长期维护。

每套主题不是单一颜色，而是一组语义调色：

- `accent`：主要强调和选中状态。
- `accentEmphasis`：强调色上的高对比图标或文字。
- `accentSoft`：弱强调背景。
- `textOnAccent`：直接放在强调色表面上的文字。
- `courseCards`：由主题基色生成的一组稳定课程卡片颜色。

使用规则：

- 不在功能页面硬编码品牌色。
- 自定义颜色必须通过对比度适配逻辑生成 `textOnAccent`。
- 颜色不能成为唯一状态编码；错误、选中和进度同时使用文字、图标或形状。
- 课程颜色用于区分，不代表课程类型、重要程度或成绩。

### 2.2 中性色与状态色

| Token | 当前语义 |
|---|---|
| `background` | 页面分组背景 |
| `backgroundSecondary` | 次级分组背景 |
| `card` / `cardBackground` | 常规内容卡片 |
| `cardElevated` | 需要轻微抬升的表面 |
| `fill` | 输入、占位或低层级填充 |
| `separator` | 低对比边界 |
| `primaryText` | 主内容 |
| `secondaryText` | 辅助说明 |
| `tertiaryText` | 占位、元信息和弱提示 |
| `warning` | 需要注意但尚未失败 |
| `danger` / `destructive` | 错误、删除和不可逆动作 |

成功状态通常使用当前 `accent` 或系统语义，不另造一套高饱和绿色。

### 2.3 圆角

| Token | 值 | 适用范围 |
|---|---:|---|
| `AppRadius.small` | 12 pt | 图标底板、紧凑块、小型状态容器 |
| `AppRadius.medium` | 16 pt | 中型卡片、输入组、按钮组 |
| `AppRadius.large` | 24 pt | 主要卡片和大容器 |

优先使用 `.continuous` 圆角。圆角层级取决于容器规模，不因页面不同而随意变化。

### 2.4 间距

| Token | 值 | 典型用途 |
|---|---:|---|
| `AppSpacing.micro` | 8 pt | 图标与文字、紧凑元信息 |
| `AppSpacing.compact` | 12 pt | 控件内部、紧凑列表项 |
| `AppSpacing.card` | 16 pt | 卡片内部或卡片之间 |
| `AppSpacing.page` | 20 pt | 页面水平边距 |
| `AppSpacing.section` | 24 pt | 主要 section 间距 |
| `AppSpacing.rootTop` | 16 pt | 根页面顶部起始间距 |

优先从令牌组合，不新增 13、17、19 等缺乏语义的局部常量。高密度课表可使用经过说明的专用布局常量。

### 2.5 阴影与边框

常规卡片：

- 1 pt `separator` 描边。
- 浅色模式使用黑色 5%、半径 10、Y 偏移 2 的轻阴影。
- 深色模式不使用黑色阴影，依靠系统底色和边界形成层级。

避免：

- 同时使用高对比边框、强阴影和渐变。
- 每个列表项都做悬浮卡片。
- 在卡片内部继续嵌套视觉权重相同的卡片。

## 3. 字体与文本

MyLeafy 当前统一使用系统字体，不依赖自定义品牌字体。字体通过 `leafyFontScale` 适配用户选择的显示密度。

| Modifier | 基础字号 | 字重 | 用途 |
|---|---:|---|---|
| `responsiveLargeTitle()` / `leafyBrandLargeTitle()` | 32 | Bold | 根页面或品牌标题 |
| `title1()` | 26 | Semibold | 重要二级标题 |
| `title2()` | 20 | Semibold | section 或主要卡片标题 |
| `leafyTitle3()` | 18 | Semibold | 小型标题 |
| `leafyHeadline()` | 16 | Semibold | 行标题、按钮重点 |
| `leafyBody()` | 16 | Regular | 正文 |
| `leafySubheadline()` | 14 | Regular | 辅助信息 |
| `microCaption()` | 11 | Regular | 时间、标签和弱元信息 |

规则：

- 优先使用语义 modifier 或 SwiftUI 文本样式，不在普通页面散落固定字号。
- 数字密集视图可使用等宽数字，但不能整页使用等宽字体。
- 正文保持自然换行，不通过缩到极小字号容纳过多信息。
- 重要按钮和错误信息不只使用 `microCaption()`。
- 英文缩写、学期编号和日期格式在多语言环境中保持可读。

## 4. 显示密度

`AppDisplaySizePreference` 提供四级显示密度：

| 选项 | 字体缩放 | 控件缩放 | 最小列表行高 |
|---|---:|---:|---:|
| 紧凑 | 0.88 | 0.88 | 40 pt |
| 标准 | 0.94 | 0.94 | 42 pt |
| 舒适 | 1.00 | 1.00 | 44 pt |
| 宽松 | 1.08 | 1.06 | 48 pt |

显示密度是 App 内整体偏好，不等同于关闭系统可访问性字体。页面必须同时考虑系统 Dynamic Type、App 密度和小屏设备。

## 5. 页面表面

### 5.1 页面背景

根页面使用 `LeafyPageBackground()`：系统 grouped background 叠加当前主题色的低透明度渐变。背景只提供轻微氛围，不应影响正文对比度。

滚动内容通常透明地覆盖在背景上；系统 `List` 页面可使用其原生分组背景，不强行套自定义 ScrollView。

### 5.2 卡片

常规卡片使用 `.leafyCardStyle()`。适用于：

- 重复内容项。
- 功能入口。
- 状态说明。
- 需要与页面背景分离的一组数据。

不适用于：

- 给单个短文本增加装饰。
- 把完整页面再包成一张大卡片。
- 系统列表已经提供分组层级的场景。

### 5.3 Glass surface

`leafyGlassSurface(in:tint:fallbackFill:isInteractive:)` 统一处理：

- iOS 26：系统 `glassEffect`。
- 较低系统：material 或明确的 fallback fill，并增加弱分隔线。

Glass 只用于工具栏、浮动控件、胶囊或少量导航表面。不要把长正文、表单和密集列表整体做成玻璃。

## 6. 核心组件

### 6.1 图标

- 优先使用 SF Symbols。
- 同类入口使用一致的 symbol 语法和 weight。
- `LeafyIconBadge` 用于带主题底色的功能图标。
- 图标必须有可访问性名称；纯装饰图标从 VoiceOver 树中隐藏。
- 不用 emoji 代替产品功能图标。

### 6.2 按钮

| 类型 | 使用场景 |
|---|---|
| 主按钮 | 登录、保存、发布等页面主要完成动作 |
| 次按钮 | 取消、稍后、辅助设置 |
| 图标按钮 | 刷新、筛选、通知、分享等熟悉的工具动作 |
| 危险按钮 | 退出、删除、撤销授权等需要确认的动作 |

`LeafyGlassIconButton` 用于顶部或浮动工具动作。单独的图标按钮必须有 accessibility label，且最小触控面积不小于 44 × 44 pt；视觉图形可以小于触控区域。

请求进行中时：

- 禁止重复提交。
- 保留按钮宽度，避免布局跳动。
- 使用进度或状态文字说明正在做什么。

### 6.3 Chip 与胶囊

`.leafyCapsuleChipSurface(isSelected:)` 用于筛选或一级领域选择：

- 选中项使用主题色和高对比文字。
- 未选中项使用抬升卡片底色和弱边界。
- 可横向滚动，但首个选项和当前选项应容易发现。
- Chip 不承担长句或危险操作。

### 6.4 表单

- 字段标签保持可见，不只依赖 placeholder。
- 验证信息贴近对应字段。
- 密码、验证码和敏感输入明确键盘类型与自动填充行为。
- 表单提交错误优先使用内嵌摘要或单个 alert，不在每个字段重复同一错误。
- 键盘出现时主要按钮和当前字段仍可到达。

### 6.5 列表与行

- 设置型信息优先使用系统 `List` 与 `insetGrouped`。
- 内容 Feed 使用自定义滚动卡片，但保持稳定的间距和加载策略。
- 行内动作数量应受控；次要动作进入 menu 或详情页。
- Swipe action 只用于用户熟悉且可恢复的动作。

## 7. 导航

### 7.1 根 Tab

- 使用系统 `TabView`。
- 标签为 `Leafy / 课表 / 社区 / 学业 / 我的`，社区可按能力隐藏。
- 选中项使用当前主题色，未选中项保持系统对比。
- 每项使用短标题和 SF Symbol。
- iOS 26 使用系统 Tab API 与视觉行为；低版本使用兼容的 `tabItem` 实现。

不为单个二级工具增加根 Tab，也不通过自定义浮动导航模拟已有系统行为。

### 7.2 层级导航

- 需要完整上下文的详情使用 `NavigationStack`。
- 轻量编辑、筛选和预览使用 sheet。
- sheet 默认根据内容使用 medium / large detents；长文阅读或复杂编辑使用全屏。
- 关闭、返回和取消使用系统位置与语义。
- 跨 Tab 跳转通过 `AppNavigationCoordinator`，页面不直接持有其他根页面。

## 8. 状态与反馈

### 8.1 Loading

- 首次加载：使用 skeleton、`ProgressView` 或明确加载文案。
- 局部刷新：已有内容继续可读，刷新控件显示进行中。
- 超过用户预期的任务说明当前阶段，例如“正在连接学校教务”。

### 8.2 Empty

空状态必须说明：

1. 什么为空。
2. 为什么可能为空。
3. 用户下一步可以做什么。

例如“本周没有课程”与“课表解析失败”必须是两个状态。

### 8.3 Error

- 使用用户可理解、可行动的描述。
- 学校网络、登录态、解析、Supabase 配置和权限错误分别处理。
- 可重试错误提供重试按钮。
- 需要开发定位时显示短 request ID，不显示内部堆栈。
- `LeafyOperationAlert` 适合单次操作结果，不用于持续页面故障。

### 8.4 Success

成功反馈应与动作风险匹配：

- 即时、可见的状态变化不必额外弹窗。
- 发布、保存或授权完成可用短反馈。
- 影响重要数据的操作应在结果中说明对象和范围。

## 9. 页面模式

### 9.1 登录

- 视觉焦点是身份表单和主操作。
- 验证码刷新和密码显隐是辅助动作。
- 学校网络与隐私说明保持简洁，不压过登录任务。

### 9.2 课表

- 网格对齐、课程名称和当前时间优先于装饰。
- 课程块在不同显示密度下仍可点击。
- 重叠课程不能遮挡。
- 背景图片必须自动控制对比度或提供遮罩。
- 课程详情在 sheet 中承接完整信息和次级动作。

### 9.3 社区

- 作者、时间、内容和互动形成稳定阅读顺序。
- 置顶、公告、投票和普通帖子有明确但克制的类型标记。
- 图片使用固定占位和渐进加载，避免滚动时布局跳动。
- 发布入口可发现，但不压过阅读。

### 9.4 学业

- 一级领域使用横向 chip 导航。
- 每个领域用 section 和工具入口组织，避免巨大图标宫格。
- 数据页显示学期、更新时间和来源。
- 图表与统计同时提供文字摘要和数值明细。

### 9.5 我的

- 使用系统设置式层级。
- 个人资料置顶，功能、外观、数据和支持分组。
- 退出和删除单独成组并使用危险色。

### 9.6 Leafy AI

- 对话列表保持低装饰，让阅读成为焦点。
- 模型状态和数据来源可被用户发现。
- Artifact 使用完成品卡片，不在消息气泡中渲染整份复杂文档。
- 流式输出过程中保留停止和错误恢复能力。

## 10. 动效

- 动效解释状态变化，不用于拖延或装饰所有交互。
- 页面切换优先使用系统导航动画。
- 列表插入、状态切换和进度变化使用短、可中断动画。
- 遵守“减少动态效果”设置；关闭非必要缩放、视差和连续背景动画。
- 加载动画不能掩盖已经失败的请求。

## 11. 可访问性

每个新增页面必须检查：

- VoiceOver 阅读顺序与视觉顺序一致。
- 图标按钮、课程块、评分和图表具有可理解标签。
- 颜色不是唯一信息载体。
- 文本放大后主要操作仍可到达，关键文本不截断。
- 触控目标至少 44 × 44 pt。
- 浅色、深色和自定义主题下保持对比度。
- 动态内容更新不造成无意义的重复播报。
- 图表提供替代摘要，评分星级同时有数值表达。

## 12. 国际化与内容适配

- 用户可见文本通过 `L10n` 或本地化资源，不在 View 中拼接不可翻译句子。
- 布局允许中文、英文和繁体中文长度变化。
- 日期、数字和学期编号使用明确格式，不依赖模糊缩写。
- 按钮标题尽量是动词，避免只写“确定”。
- 不通过固定 frame 为单一语言“调像素”。

## 13. 设计评审清单

提交 UI 变更前检查：

- 是否复用了 `AppTheme`、`AppSpacing`、`AppRadius` 和 `AppChrome`。
- 页面层级是否可以用更少的卡片或更原生的系统结构表达。
- 所有远程数据是否有 Loading、Empty、Error 和恢复状态。
- 深色模式、三套预设主题和自定义主题是否可读。
- 四档显示密度、Dynamic Type 和小屏是否可用。
- VoiceOver 标签、阅读顺序和触控面积是否合格。
- iOS 17 回退与 iOS 26 增强是否都经过检查。
- 是否避免了硬编码颜色、无语义间距、emoji 图标和工程化错误文案。
