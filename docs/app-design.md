# MyLeafy App 设计

本文档记录当前 App 的产品结构、页面状态和主要交互。架构实现见 [架构说明](architecture.md)，视觉细节见 [UI 风格指南](ui-style-guide.md)。

## 1. 产品定位

MyLeafy 是以课表为核心的通用型校园工具 App；当前首个上线校园是北京林业大学。设计重点是快速打开、快速确认今天安排，并把社区作为主入口，同时把成绩、考试、教学培养、自习室、校历和评教集中到稳定的学业入口中。

当前产品原则：

- 根入口为 `课表 / 社区 / 学业 / 我的`。
- 教务数据优先直连学校强智系统。
- 社区、通知、反馈、评教等非教务业务由 Supabase 承接。
- 页面尽量使用 SwiftUI 原生导航、列表、sheet、menu 和 alert。
- 教务相关复杂工具收进学业页，社区保持独立主入口。

## 2. 全局信息架构

启动逻辑：

1. 无学校身份缓存时进入登录页。
2. 登录成功后进入根 Tab。
3. 已登录冷启动时直接进入根 Tab。

根 Tab：

- `课表`：默认核心页，展示当前周和当天课程。
- `社区`：承接帖子流、发布、通知、公告和社区互动。
- `学业`：承接教务工具、校园日程和评教。
- `我的`：承接社区资料、个人内容、收藏、设置、支持和退出登录。

## 3. 登录

登录页字段固定为：

- 学号
- 密码
- 验证码

交互：

- 验证码图片可刷新。
- 密码支持显隐切换。
- 登录按钮在请求中展示加载态。
- 登录失败使用系统 alert，优先展示学校返回的可读错误。

状态：

- 验证码加载中。
- 登录中。
- 登录失败。
- 会话建立成功。

登录成功后，App 保存学校身份，并在需要社区功能时再初始化 Supabase 匿名会话和社区 profile。

## 4. 课表

课表是默认首页。页面由自定义头部和周视图网格组成：

- 顶部展示标题、当前周日期范围、周次切换和刷新按钮。
- 支持按周过滤课程。
- 课程块按节次占位展示。
- 同日重叠课程并排压缩，不直接覆盖。
- 点击课程打开半屏详情。

课程详情字段：

- 课程名
- 教师
- 教室
- 周次
- 节次

当前还包含日程辅助体验：

- 今日课程摘要。
- 单日课程列表。
- 分享图预览和系统分享。
- 天气信息加载。
- 自习室查询跳转。

状态：

- 未登录。
- 首次加载中。
- 抓取失败。
- 解析失败。
- 当前周无课。
- 成功展示课程。

## 5. 社区和学业

社区页顶部保留天气、筛选、通知和发布入口。学业页直接使用横向胶囊一级导航。

学业一级导航：

- `成绩`
- `教学培养`
- `自习室`
- `校历`
- `评教`

### 社区

社区当前已接 Supabase，不再是纯 Mock：

- 帖子流。
- 全局置顶和分类置顶帖子展示。
- 发文本或带图帖子。
- 帖子详情。
- 一级评论。
- 点赞和取消点赞。
- 我的发帖、我的点赞、我的评论。
- 发帖、评论、点赞前要求完成社区昵称。
- 删除自己的帖子或评论。

发布入口位于社区页右上角。未完成社区资料时，先打开资料编辑 sheet。

### 通知和公告

通知入口位于社区页右上角：

- 未读数以红点提示。
- 评论和点赞通知可打开对应帖子。
- 站内公告可打开公告详情。
- 支持全部静音设置。
- 支持单条通知/公告处理。

### 成绩

成绩页承接真实教务直连：

- 成绩查询。
- 考试安排。
- 成绩分析和排名相关视图。

失败状态需要区分网络问题、学校登录态问题和解析问题。

### 教学培养

教学培养页包含：

- 教学计划。
- 培养方案。
- 毕业学分进度。

数据来自学校教务页面解析，展示时按学期、课程和学分组织。

### 自习室

自习室入口用于按条件查询可用教室：

- 楼宇。
- 日期。
- 起止节次。
- 指定教室占用。
- 常用自习室收藏。

课表或收藏入口可以请求学业页打开指定自习室查询。

### 校历

校历页承接本地资源和本地日程：

- 校历图片。
- 作息时间图片。
- 校园日程。
- 自定义倒计时。

校历和作息图支持全屏预览、左右切换、缩放和拖拽。

### 评教

评教当前已接 Supabase 老师名录和星级评分：

- 搜索教师或学院。
- 教师卡片展示平均分、评分数和星级分布。
- 教师详情页提交或更新 1 到 5 星评分。
- 当前只统计星级，不支持文字评价。

教师名录为空时，页面提示先导入 `teachers` 表 CSV。

## 6. 我的

我的页使用系统 `List + insetGrouped` 风格，结构为：

- `资料`
- `内容`
- `设置`
- `支持`
- `退出登录`

### 资料

资料区展示社区 profile：

- 头像。
- 昵称或默认资料标题。
- 教务学号绑定说明。
- 专业和年级。

点击资料区进入社区资料编辑 sheet。昵称必填；头像、专业、年级可选。

### 内容

内容区包含：

- 我的发帖。
- 我的点赞。
- 我的评论。
- 常用链接。
- 我的收藏，用于查看私密保存的社区帖子。

### 设置

设置区包含：

- 主题色。
- 深色模式。

主题色当前支持鼠尾草绿、蒂芙尼蓝、阳光琥珀、珊瑚粉红、蜜橘橙、柠檬黄、青柠绿、海岛晴蓝、糖果莓粉、兰花紫。深色模式可强制开启；关闭时跟随系统。

### 支持

支持区包含：

- 意见反馈。
- 联系我们。
- 缓存与同步。
- 数据安全。
- 关于 MyLeafy。

意见反馈写入 Supabase；联系我们展示反馈群二维码资源。

### 退出登录

退出登录使用红色危险操作样式，并弹出二次确认。退出后清理学校会话和社区会话，本地课表与成绩缓存保留。

## 7. 状态设计原则

每个核心页面需要覆盖：

- `Loading`：明确当前正在加载学校数据、社区数据还是本地资源。
- `Empty`：给出下一步动作，例如刷新、导入教师名录、完善资料。
- `Error`：错误文案面向用户，不直接暴露 Cookie 或原始调试信息。
- `Unauthenticated`：学校会话失效时要求重新登录；社区会话缺失时尝试重新 bootstrap。

错误提示要能区分：

- 学校网络不可达。
- 学校登录态过期。
- HTML 结构变化导致解析失败。
- Supabase 配置缺失。
- Supabase RLS 或 Edge Function 拒绝操作。
